# 日常命令

本文档记录 Codex 多账号管理器的本地开发、验证和发行命令。

## 运行程序

优先使用根目录脚本：

```bat
run.bat
```

如果 `dist\CodexAccountManager-win-x64\CodexAccountManager.exe` 不存在，`run.bat` 会先自动执行一次打包发布。

## 构建与测试

运行完整的还原、测试和构建：

```bat
scripts\build-and-test.bat
```

等价核心命令：

```bat
dotnet restore CodexAccountManager.sln
dotnet test tests\CodexAccountManager.Core.Tests\CodexAccountManager.Core.Tests.csproj
dotnet build CodexAccountManager.sln -p:Platform=x64 --no-restore
```

## 打包发行

日常打包使用根目录脚本：

```bat
package.bat
```

底层发布脚本：

```bat
scripts\publish-portable.bat
```

发布脚本会依次执行：

```bat
dotnet restore CodexAccountManager.sln
dotnet test tests\CodexAccountManager.Core.Tests\CodexAccountManager.Core.Tests.csproj --no-restore
dotnet publish src\CodexAccountManager.App\CodexAccountManager.App.csproj -c Release -p:Platform=x64 -r win-x64 --self-contained true -o dist\CodexAccountManager-win-x64
```

然后压缩便携发行包：

```text
dist\CodexAccountManager-win-x64-portable.zip
```

可执行文件位置：

```text
dist\CodexAccountManager-win-x64\CodexAccountManager.exe
```

## 什么时候需要重新打包

需要重新打包：

- 改了 C# 代码、XAML UI、图标、图片或其他资源文件。
- 改了项目文件、依赖、发布配置或应用清单。
- 要把最新版本交给别人使用，或替换 `dist` 里的正式发行包。

通常不需要重新打包：

- 只改 README、设计文档、开发记录等不随程序运行的文档。
- 只在本机 Debug 调试，且不需要更新 `dist` 发行目录。

## 清理输出

需要手动清理发行输出时，可删除：

```text
dist\CodexAccountManager-win-x64
dist\CodexAccountManager-win-x64-portable.zip
```

随后重新运行：

```bat
package.bat
```
